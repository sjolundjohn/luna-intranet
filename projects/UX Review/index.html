The design-system foundation for the Luna iOS app review platform: brand guidelines, the iOS component board, and the canonical two-step dose-confirmation component. Every wireframe and developer build references this kit.

01 · Reference

Brand Guidelines

Logo usage, color palette + guardrails, the 11-level iOS HIG type scale in D-DIN, spacing, radii, elevation, voice & imagery.
02 · Library

iOS Component Board

Status card, glucose display, CGM chart, buttons, list rows, tab bar, toast, insight card, calendar strip, bottom-sheet modal, nav header, setup cards.
03 · Safety-critical

Dose-Confirmation Component

The canonical two-step, two-part-action confirmation: acknowledge both required actions, then slide to confirm. All six states, interactive.

About this kit

This is the Luna iOS app design system — intentionally distinct from the nightluna intranet's own styling (D-DIN everywhere, iOS HIG type scale, larger radii). Tokens live in css/tokens.css; raw reference cards in preview/; brand assets in assets/; fonts in fonts/.

  • Portable & offline: no CDN or build step — open any .html directly.
  • Ports to Astro: the UX Review section of nightluna scopes these tokens to the wireframe/component render area (separate from intranet chrome) and re-implements ios-kit components + the dose component natively. See INFRASTRUCTURE-BRIEF.md.
  • Figma-ready: component names here = future Figma layer names.
